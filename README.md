# 八字國考分析

## 計算架構

專案目前分成三層：

1. 曆法核心：農曆轉換、節氣四柱、藏干、十神、起運與大運。
2. 旺衰模型：以月令的旺相休囚死做透明權重試算。
3. 國考觀察：把十神群組與旺衰試算轉成讀書提醒，不輸出錄取機率。

## 離線資料庫

完整清單：`data/DATABASE-CATALOG.md`

- `DB-001 國考職群職能矩陣`
- `DB-002 八字五行與十神解讀規則`
- `DB-003 個人偏好與情緒校正規則`
- `DB-004 情緒與壓力提醒庫`（30 條）
- `DB-005 讀書策略提醒庫`（30 條）
- `DB-006 考場風險提醒庫`（30 條）
- `DB-007 職務環境適應提醒庫`（30 條）

職群排序使用固定公式：

- 五行匹配：40%
- 職能匹配：35%
- 題型匹配：15%
- 工作環境：10%

輸出的分數是資料庫內的相對匹配度，不是錄取率。

## 旺衰模型

旺衰不是跨流派一致的單一公式。目前採用明載規則：

- 明干權重：1
- 藏干本氣／中氣／餘氣：1／0.6／0.3
- 旺／相／休／囚／死季節係數：1.5／1.25／1／0.75／0.5
- 同五行與生扶日主的五行合計為支持比例

畫面會標示這是試算模型，不將它冒充為唯一的格局或喜用神定論。

## 限制

- 目前時間基準固定為台灣標準時間 UTC+8。
- 尚未加入出生地經緯度與真太陽時修正。
- 國考判讀沒有統計錄取資料支持，只能作傳統文化角度的輔助描述。
- 要建立可驗證的國考模型，還需匿名收集考試類型、科目、準備時數、模考成績與實際結果。
